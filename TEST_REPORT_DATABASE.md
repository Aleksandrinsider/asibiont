## ⚠️ Найденные проблемы

### 1. КРИТИЧЕСКАЯ: Timezone теряется при сохранении в БД

**Симптомы:**
- Все задачи сохраняются с `reminder_time.tzinfo = None`
- Даже если `add_task` конвертирует в UTC, после чтения из БД tzinfo = None

**Причина:**
- SQLite (используется в LOCAL=1 режиме) НЕ поддерживает timezone-aware datetime
- Column определено как `DateTime` вместо `DateTime(timezone=True)`

**Влияние:**
- В локальной разработке - минимальное (можно считать все времена UTC)
- В продакшене с PostgreSQL - может привести к проблемам если не использовать `DateTime(timezone=True)`

**Решение:**
```python
# В models.py:
reminder_time = Column(DateTime(timezone=True), index=True)  # Для PostgreSQL
due_date = Column(DateTime(timezone=True))
actual_completion_time = Column(DateTime(timezone=True))
created_at = Column(DateTime(timezone=True), default=lambda: datetime.now(timezone.utc))
```
