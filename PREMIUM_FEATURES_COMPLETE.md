#### 🏃 `set_activity_alert` - Уведомления об активностях
```python

"Скажи когда кто-то пойдет на пробежку"
→ AI следит за задачами других
→ Когда кто-то создает задачу "пробежка завтра в 7:00"
→ AI естественно упоминает в разговоре: "Кстати, @user123 планирует пробежку завтра..."
```

**Параметры:**
- `activity_type`: тип активности (пробежка, митап, воркшоп)
- `keywords`: ключевые слова для поиска
- `location`: фильтр по городу (опционально)
- `frequency`: 'any', 'regular', 'one_time'

#### 👥 `set_contact_alert` - Уведомления о новых специалистах
```python
# Пример использования:
"Предупреди когда появится специалист по продажам"
→ AI следит за регистрациями
→ Когда регистрируется нужный специалист
→ AI естественно упоминает: "Появился новый специалист по продажам @user456..."
```

**Параметры:**
- `skill`: навык для поиска (продажи, Python, дизайн)
- `interest`: интерес для поиска (стартапы, ИИ, блокчейн)
- `city`: фильтр по городу (опционально)
- `position`: фильтр по должности (опционально)

## 🎯 Как это работает

### Workflow:

1. **Настройка алерта** (через AI диалог):
   ```
   User: "Скажи когда кто-то пойдет на пробежку"
   AI: вызывает set_activity_alert(type="пробежка", keywords=["пробежка"])
   AI: "✅ Настроил! Буду следить..."
   ```

2. **Мониторинг** (автоматически):
   - Activity: проверяет новые задачи за последние 24 часа
   - Contact: проверяет обновления профилей за последние 24 часа

3. **Триггер** (когда находит совпадение):
   - Добавляет в контекст: `🔔 @user123 планирует: пробежка в 7:00`
   - Обновляет `last_triggered_at`

4. **AI упоминает** (естественно в разговоре):
   ```
   User: "Привет, как дела?"
   AI: "Привет! Кстати, @user123 планирует пробежку завтра в 7:00, может присоединишься?"
   ```

### Ключевые особенности:

✅ **Проактивный контекст** - информация приходит через промпт, НЕ через push
✅ **Естественная интеграция** - AI сам решает когда упомянуть
✅ **Premium only** - доступно только для Premium подписки
✅ **Автоматический мониторинг** - без участия пользователя
✅ **Без навязчивости** - никаких лишних уведомлений
