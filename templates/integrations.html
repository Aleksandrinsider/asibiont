{% if lang == 'en' %}Incoming Webhooks{% else %}Входящие вебхуки{% endif %}

{% if lang == 'en' %}Connect any external service to your AI agent — CRM, GitHub, payment systems, your own server{% else %}Подключите любой внешний сервис к AI-агенту — CRM, GitHub, платёжные системы, ваш сервер{% endif %}

{% if lang == 'en' %}What are incoming webhooks?{% else %}Что такое входящие вебхуки?{% endif %}
{% if lang == 'en' %} Any external service can send events to your AI agent. The agent receives the payload and sends you a notification in Telegram. GitHub can notify about a new pull request, your CRM about a new lead, your payment system about a successful payment — and the agent immediately creates a task or sends a message. {% else %} Любой внешний сервис может отправлять события вашему AI-агенту. Агент получает данные и присылает вам уведомление в Telegram. GitHub сообщит о новом pull request, CRM — о новом лиде, платёжная система — об успешной оплате, и агент тут же создаст задачу или отправит сообщение. {% endif %}
{% if lang == 'en' %}How to set up a webhook?{% else %}Как настроить вебхук?{% endif %}
  1. {% if lang == 'en' %}Find your agent ID in the agent settings in the dashboard.{% else %}Найдите ID вашего агента в настройках агента в дашборде.{% endif %}
  2. {% if lang == 'en' %}Get a webhook token via API:{% else %}Получите токен вебхука через API:{% endif %} 
    GET https://asibiont.com/api/agent-webhook-token/{agent_id}?action=regenerate
    {% if lang == 'en' %}Response:{% else %}Ответ:{% endif %} 
    {"token": "abc123...", "url": "https://asibiont.com/api/agent-webhook/42/abc123..."}
  3. {% if lang == 'en' %}Configure the external service to send POST requests to:{% else %}Настройте внешний сервис отправлять POST-запросы на:{% endif %} 
    POST https://asibiont.com/api/agent-webhook/{agent_id}/{token}
    {% if lang == 'en' %}Request body (JSON, any fields):{% else %}Тело запроса (JSON, любые поля):{% endif %} 
    {
  "event": "new_order",
  "order_id": 12345,
  "amount": 9900,
  "customer": "Иван Петров"
}
  4. {% if lang == 'en' %}The agent receives the event, creates a task, and sends you a notification in Telegram.{% else %}Агент получит событие, создаст задачу и пришлёт вам уведомление в Telegram.{% endif %}
{% if lang == 'en' %}Is the webhook secure?{% else %}Насколько это безопасно?{% endif %}
{% if lang == 'en' %} The token is verified via constant-time comparison (timing-safe) — this prevents timing attacks. You can regenerate the token at any time by calling the same endpoint with ?action=regenerate. The old token becomes invalid immediately. {% else %} Токен проверяется через защищённое сравнение (timing-safe) — это исключает timing-атаки. Вы можете перегенерировать токен в любой момент, снова вызвав endpoint с ?action=regenerate. Старый токен сразу становится недействительным. {% endif %}
{% if lang == 'en' %}Which services can send webhooks?{% else %}Какие сервисы можно подключить?{% endif %}
{% if lang == 'en' %}Any service that supports outgoing webhooks (POST requests):{% else %}Любой сервис, поддерживающий исходящие вебхуки (POST-запросы):{% endif %}
  • GitHub / GitLab — {% if lang == 'en' %}pull requests, issues, deployments{% else %}pull requests, issues, деплои{% endif %}
  • {% if lang == 'en' %}Payment systems{% else %}Платёжные системы{% endif %} (Stripe, ЮKassa, Robokassa)
  • {% if lang == 'en' %}CRM systems{% else %}CRM-системы{% endif %} (AmoCRM, Bitrix24, HubSpot)
  • {% if lang == 'en' %}Your own server or script{% else %}Ваш собственный сервер или скрипт{% endif %}
  • Zapier, Make (Integromat), n8n
  • {% if lang == 'en' %}Any monitoring or alert system{% else %}Любая система мониторинга или алертов{% endif %}
{% if lang == 'en' %}What format does the request body need to be?{% else %}Какой формат тела запроса нужен?{% endif %}
{% if lang == 'en' %} JSON with any fields. The agent receives the entire payload and converts it into a readable notification. There are no required fields — send whatever your service provides. The Content-Type header should be application/json. {% else %} JSON с любыми полями. Агент получит весь payload и преобразует его в читаемое уведомление. Обязательных полей нет — можно отправлять то, что предоставляет ваш сервис. Заголовок Content-Type должен быть application/json. {% endif %}
