# Полная цепочка делегирования задач

## Текущая реализация (обновлено 29.01.2026)

### 1. Создание и делегирование задачи

**Делегатор (пользователь A):**
- Через AI чат: "поручи @username задачу..."
- `delegate_task()` создает задачу с полями:
  - `user_id` = ID делегатора (владелец задачи)
  - `delegated_by` = ID делегатора
  - `delegated_to_username` = username исполнителя
  - `delegation_status` = "pending"

**Исполнитель (пользователь B) получает уведомление:**
- Бот отправляет сообщение о новой делегированной задаче
- В дашборде задача появляется в разделе "Поручили мне"

---

### 2. Принятие/Отклонение задачи

**Исполнитель может:**
- ✅ Принять задачу → `delegation_status` = "accepted"
- ❌ Отклонить задачу → `delegation_status` = "rejected"

**Фильтрация в дашборде:**
- "Поручили мне": `task.delegated_by != user.id AND task.delegation_status == 'accepted'`
- "Поручил я": `task.delegated_by == user.id`

---

### 3. Выполнение делегированной задачи ⭐ НОВОЕ

**Когда исполнитель завершает задачу:**

#### Через кнопку дашборда:
1. `completeTask(taskId)` → `/complete_task` endpoint
2. Backend вызывает `complete_task()` из `ai_integration/handlers.py`
3. **ЛОГИКА ОПРЕДЕЛЕНИЯ ДЕЛЕГИРОВАНИЯ:**
   ```python
   # Случай 1: Задача делегирована МНЕ
   if task.delegated_by and task.delegated_by != user.id and task.delegation_status == "accepted":
       delegator = session.query(User).filter_by(id=task.delegated_by).first()
       is_delegated_task = True
   
   # Случай 2: Задача делегирована МНОЙ (не должно происходить при выполнении)
   elif task.user_id == user.id and task.delegated_to_username:
       # Делегатор не может завершить задачу сам
   ```

4. **ЗАПРОС РЕЗУЛЬТАТОВ У ИСПОЛНИТЕЛЯ:**
   ```python
   if is_delegated_task and delegator:
       # Отправляем запрос результатов исполнителю
       result_request = f"📝 Расскажи о результатах выполнения задачи: '{task.title}'"
       await bot.send_message(chat_id=user.telegram_id, text=result_request)
       
       # Сохраняем telegram_id делегатора для отправки отчета
       task.pending_delegator_report = delegator.telegram_id
       session.commit()
   ```

#### Через AI чат:
1. Исполнитель пишет: "я выполнил задачу X"
2. AI вызывает `complete_task()` → та же логика выше
3. Бот запрашивает результаты

---

### 4. Сбор и отправка результатов ⭐ НОВОЕ

**В `ai_integration/chat.py` добавлена проверка ПЕРЕД обработкой AI:**

```python
# Проверяем есть ли задачи, ожидающие отчета
pending_report_tasks = db_session.query(Task).filter(
    Task.pending_delegator_report.isnot(None),
    Task.user_id == user.id,
    Task.status == 'completed'
).all()

if pending_report_tasks:
    task = pending_report_tasks[0]
    delegator_telegram_id = task.pending_delegator_report
    
    # Сохраняем результаты
    task.completion_notes = encrypt_data(clean_message)
    task.pending_delegator_report = None  # Убираем флаг
    db_session.commit()
    
    # Отправляем отчет делегатору
    report_message = f"""
    📋 Отчет о выполнении задачи
    
    👤 Исполнитель: @{user.username}
    📝 Задача: {task.title}
    ✅ Статус: Выполнена
    📄 Результаты:
    {clean_message}
    
    Оцени качество выполнения или задай вопросы исполнителю.
    """
    await bot.send_message(chat_id=delegator_telegram_id, text=report_message)
    
    # Подтверждение исполнителю
    return f"✅ Я отправил отчет @{delegator.username}. Спасибо за работу!"
```

**Сценарий:**
1. Исполнитель завершает задачу → получает запрос результатов
2. Исполнитель пишет в чат описание результатов → система перехватывает
3. Система сохраняет результаты в `completion_notes`
4. Система отправляет отчет делегатору в Telegram
5. Исполнитель получает подтверждение

## Новые поля в модели Task

```python
class Task(Base):
    # ... существующие поля ...
    
    # Новое поле для отслеживания ожидания отчета
    pending_delegator_report = Column(BigInteger)  # telegram_id делегатора
```

**Логика:**
- `pending_delegator_report IS NULL` → отчет не ожидается
- `pending_delegator_report = 123456789` → ожидается отчет, будет отправлен пользователю с этим telegram_id

## Что гарантируется

### ✅ Делегатор ГАРАНТИРОВАННО получит:
1. **Уведомление о принятии/отклонении** задачи исполнителем
2. **Запрос результатов** у исполнителя при завершении задачи
3. **Полный отчет** с описанием результатов работы
4. **Возможность оценить** качество выполнения (в будущем)

### ✅ Исполнитель ГАРАНТИРОВАННО:
1. Получит уведомление о новой делегированной задаче
2. Сможет принять или отклонить задачу
3. При завершении получит запрос описания результатов
4. Получит подтверждение об отправке отчета

### ✅ AI агент ГАРАНТИРОВАННО:
1. Перехватит следующее сообщение от исполнителя после завершения
2. Отправит отчет делегатору в Telegram
3. Сохранит результаты в зашифрованном виде (`completion_notes`)
4. Очистит флаг ожидания (`pending_delegator_report = NULL`)

## Известные ограничения

### ⚠️ НЕ реализовано:
1. **Оценка качества работы** — делегатор пока не может оценить результат
2. **Запрос доработки** — нельзя вернуть задачу на доработку
3. **История отчетов** — нельзя посмотреть старые отчеты в дашборде
4. **Множественные исполнители** — нельзя делегировать задачу нескольким людям
5. **Частичное выполнение** — нельзя отметить прогресс (0%, 50%, 100%)

### ⚠️ Возможные проблемы:
1. **Если исполнитель не отвечает** на запрос результатов → флаг остается, следующее сообщение будет воспринято как отчет
2. **Если бот недоступен** → отчет не будет отправлен (ошибка в логах)
3. **Если делегатор удален** → отчет не будет отправлен (ошибка в логах)
