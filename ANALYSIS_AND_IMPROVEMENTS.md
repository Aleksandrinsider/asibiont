## 🔍 Обнаруженные проблемы

### 1. **КРИТИЧНО: AI не видит текущие задачи в контексте**
**Локация:** `ai_integration.py:1815-1820`
```python
# НЕ загружаем задачи в user_memory! Агент должен сам вызвать list_tasks()
# Это критично для предотвращения выдумывания задач
```

**Проблема:** AI получает только `upcoming_reminders` (5 ближайших), но НЕ видит:
- Все текущие задачи пользователя
- Просроченные задачи
- Задачи без напоминаний
- Общее количество задач

**Последствия:**
- AI не может проактивно предложить помощь с задачами
- Не видит контекст при добавлении похожих задач
- Не может определить загруженность пользователя
- Не понимает, есть ли уже аналогичная задача

**Решение:** Добавить краткую сводку задач в контекст:
```python
tasks_summary = session.query(Task).filter_by(user_id=user.id, status='pending').count()
overdue_count = session.query(Task).filter(
    Task.user_id == user.id,
    Task.reminder_time < user_now,
    Task.status == 'pending'
).count()
user_memory += f"\nСводка задач: всего активных {tasks_summary}, просрочено {overdue_count}"
```

### 2. **Уточнения к задачам не всегда обрабатываются**
**Локация:** `ai_integration.py:238-247`

**Проблема:** Инструкция есть, но:
- Нет механизма отслеживания последней добавленной задачи
- AI должен сам понять из контекста, что это уточнение
- Форсированный `add_task` не знает о предыдущей задаче

**Решение:** Добавить в user_memory ID последней добавленной задачи:
```python
# После успешного add_task сохранить в сессию:
last_task_id = task.id
user_memory += f"\nПоследняя добавленная задача: ID={last_task_id}, title='{task.title}'"
```

### 3. **Проблемы с извлечением времени из форсированного add_task**
**Локация:** `ai_integration.py:1630-1670`

**Текущие паттерны:**
- ✅ "через 5 минут" 
- ✅ "завтра в 15:00"
- ❌ "в 15:00" (без "завтра")
- ❌ "через полчаса"
- ❌ "утром" / "вечером"
- ❌ "после обеда"

**Решение:** Расширить регулярные выражения:
```python
# Добавить паттерны:
- r'в\s+(\d{1,2}):(\d{2})' для "в 15:00"
- r'через\s+(пол)?часа?' для "через полчаса"
- r'(утром|днем|вечером|ночью)' с маппингом времени
```

if "поручи" in message_lower or "делегируй" in message_lower:

    if partners:
        for partner in partners:
            name = partner['first_name'].lower()
            if name in message_lower:
                # Найти username по имени и вызвать delegate_task

### 5. **Timezone проблемы при первом запуске**
**Локация:** `ai_integration.py:1850-1865`

**Проблема:** Если у пользователя не установлен timezone:
- Используется UTC
- Но в России это +3 часа разницы
- Задачи создаются с неправильным временем

**Решение:** При создании пользователя устанавливать дефолтный timezone:
```python
# В handlers.py при регистрации:
user.timezone = 'Europe/Moscow'  # Дефолт для русскоязычных
```

if not profile_filled and len(context) < 3:  # Первые 3 сообщения

### 8. **Форсированные вызовы конфликтуют с AI вызовами**
**Локация:** `ai_integration.py:2150-2200`

**Проблема:** Если AI вызвал функцию, но форсированный механизм ТОЖЕ вызвал:
- Задача добавляется дважды
- Или обновляется когда не нужно

**Решение:** Проверять результат AI перед форсированными вызовами:
```python
if tool_calls:  # AI уже вызвал функции
    forced_calls = None  # Не форсировать
else:
    forced_calls = force_tool_calls(...)
```

### 10. **Overdue tasks не видны в контексте**
**Локация:** `ai_integration.py:1880-1895`

**Проблема:** Upcoming reminders показывает только будущие, НО:
- Просроченные задачи не видны AI
- AI не может проактивно предложить помощь
- Пользователь не получает напоминания о просрочке

**Решение:** Добавить отдельно:
```python
overdue_tasks = session.query(Task).filter(
    Task.user_id == user.id,
    Task.reminder_time < user_now,
    Task.status == 'pending'
).limit(5).all()
if overdue_tasks:
    overdue_info = [f"{t.title} (просрочена на {format_overdue_time(user_now - t.reminder_time)})" for t in overdue_tasks]
    user_memory += f"\n⚠️ ПРОСРОЧЕННЫЕ ЗАДАЧИ: {', '.join(overdue_info)}"
```
