# Архитектурные изменения: Унификация AI-агента

## Проблема
AI-агент генерировал ответы с эмодзи, жирным текстом, маркированными списками и короткие ответы вместо развернутого анализа, несмотря на правила в промпте.

## Причина
Отдельные `analysis_system` промпты для `list_tasks` конфликтовали с главным системным промптом и создавали несогласованность в поведении агента.

## Решение

### 1. Удаление конфликтующих промптов
- ❌ Убрали все отдельные `analysis_system` для `list_tasks`
- ✅ Теперь используется только **единый главный промпт** (`get_system_prompt()`)

### 2. Усиление главного промпта
Добавлены **СТРОГИЕ ПРАВИЛА ФОРМАТА ОТВЕТОВ** в начало системного промпта:

```python
def get_system_prompt():
    return """Ты — ИИ-помощник для управления задачами в Telegram.

🚨 СТРОГИЕ ПРАВИЛА ФОРМАТА ОТВЕТОВ (ОБЯЗАТЕЛЬНО СОБЛЮДАТЬ):
- НИКОГДА не используй эмодзи, смайлики, иконки
- НИКОГДА не используй жирный текст (**), курсив, заголовки
- НИКОГДА не используй маркированные списки (-, •, *) или нумерацию (1., 2., 3.)
- Всегда пиши обычным текстом без форматирования
- Минимум 3-4 предложения в каждом ответе
- Каждый ответ должен заканчиваться вопросом или предложением для продолжения диалога
- Для анализа задач: минимум 300 слов, детальный разбор каждой задачи
```

### 3. Механизм принуждения соответствия
Добавлены две новые функции:

#### `validate_response_compliance(response_text, intent_type)`
Проверяет ответ AI на соответствие правилам:
- Наличие эмодзи
- Использование жирного текста/форматирования
- Наличие списков и нумерации
- Минимальная длина ответа
- Наличие вопросов для вовлечения

#### `enforce_prompt_compliance(response_text, ...)`
Автоматически исправляет несоответствующие ответы:
- Делает повторный запрос к AI с указанием проблем
- До 2 попыток исправления
- Возвращает исправленный ответ

### 4. Интеграция в основной поток
Механизм встроен в `chat_with_ai()`:
- После генерации ответа → проверка соответствия
- При несоответствии → автоматическая коррекция
- Применяется ко всем типам ответов (tool calls и обычные)

## Преимущества

### ✅ Единство поведения
Один главный промпт для всех функций обеспечивает consistent ответы

### ✅ Автоматическая коррекция
AI сам исправляет свои ответы при нарушении правил

### ✅ Масштабируемость
Легко добавить новые правила проверки в `validate_response_compliance()`

### ✅ Прозрачность
Все правила в одном месте - легко понять и изменить поведение агента

## Архитектура

```
Пользователь → chat_with_ai()
                    ↓
         Главный промпт (get_system_prompt)
         - Строгие правила формата
         - Запреты на эмодзи/списки
         - Требования к длине
                    ↓
         AI генерирует ответ
                    ↓
         validate_response_compliance()
         - Проверка на соответствие
                    ↓
         [Не соответствует?]
                    ↓
         enforce_prompt_compliance()
         - Повторный запрос с коррекцией
                    ↓
         Финальный ответ пользователю
```
