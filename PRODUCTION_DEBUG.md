# Диагностика проблем с фильтрацией контактов

## Проблемы
1. Пользователь видит все контакты (хотя должен видеть только Bronze/Silver)
2. Стили тарифов не в формате панели GitHub
3. Контент не по центру на странице тарифов

## Возможные причины

### 1. Тарифы в production БД не установлены
Все пользователи могут иметь дефолтный тариф или одинаковые тарифы.

**Решение:** Запустить скрипт обновления тарифов на Railway:
```bash
python update_production_tiers.py
```

### 2. Код на Railway не обновился
Railway может кэшировать старую версию кода.

**Проверка:** 
- Проверить логи Railway на наличие новых коммитов
- Принудительно перезапустить приложение в Railway dashboard

### 3. Кэширование CSS/JS
Браузер или Railway кэширует старые стили.

**Решение:**
- Обновлена версия CSS до `v=1.5`
- Очистить кэш браузера (Ctrl+F5)
- Проверить, что CSS загружается: `/static/style.css?v=1.5`

## Диагностика

### Проверить тарифы пользователей
```bash
# В Railway console или через SSH:
python check_tiers.py
```

### Проверить логи сервера
Посмотреть логи Railway на наличие сообщений о фильтрации контактов.

```bash
# Локально все работает правильно
$env:LOCAL="1"; python check_tiers.py
# Должен показать: bronze: 3, silver: 2, gold: 1
```
