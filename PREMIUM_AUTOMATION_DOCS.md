# Premium Automation - Документация

## Концепция

Premium автоматизация использует AI для косвенной помощи Premium пользователям в достижении их бизнес-целей. 

**Ключевая идея**: Вместо явного делегирования задач, AI отправляет **полезные рекомендации** релевантным людям, которые выглядят как естественные советы от AI ассистента.

## Как это работает

### 1. Premium пользователь устанавливает цели в профиле

```python
goals = """
Бизнес-цели на Q1 2025:

1. Найти партнёров для дистрибуции нового продукта (EcoTech Solutions)
2. Нанять 2 Python разработчиков (Senior level) для backend команды
3. Установить связи с владельцами каналов в нише B2B SaaS
"""
```

### 2. Фоновый job (каждые 12 часов) анализирует цели через AI

- Извлекает actionable items
- Определяет критерии для поиска релевантных людей
- Формирует "возможности" для представления

### 3. AI находит релевантных пользователей

Matching по:
- **Интересам** в профиле
- **Навыкам** в профиле
- **Локации** (опционально)

Scoring: 
- 40% — совпадение интересов
- 60% — совпадение навыков
- 20% — географическая близость

### 4. AI отправляет персонализированные рекомендации

**Пример:**

Premium цель: "Найти партнёров для дистрибуции EcoTech"

AI находит пользователя с интересами "дистрибуция, оптовые продажи, экотовары"

Отправляет сообщение:
```
Привет! Вижу ты интересуешься дистрибуцией экотоваров. 
Заметил интересный продукт EcoTech Solutions — может подойти 
для твоей аудитории. Интересно?
```

**Пользователь думает:** "Полезная рекомендация от AI" ✅  
**Premium получает:** Заинтересованного партнёра ✅

## Архитектура

```
ai_integration/premium_simple.py
├── run_premium_automation()          # Основной цикл
│   ├── analyze_goals_with_ai()       # AI анализ целей
│   ├── find_relevant_users()         # Matching пользователей
│   └── send_suggestion_to_user()     # Отправка рекомендации
│
├── schedule_premium_automation()     # Настройка фонового job
└── format_premium_report()           # Отчёт для Premium
```

### Интеграция

**subscription_service.py:**
- `activate_subscription()` → `schedule_premium_automation()` если tier=PREMIUM
- `cancel_subscription()` → `unschedule_premium_automation()` если был PREMIUM

**reminder_service.py:**
- APScheduler job каждые 12 часов
- Использует `REMINDER_SERVICE.bot` для отправки

**models.py:**
- `UserProfile.goals` — хранит цели Premium пользователя
- `UserProfile.interests` — для matching
- `UserProfile.skills` — для matching

## Этика и privacy

### Прозрачность
- Пользователи знают что AI может отправлять рекомендации
- В terms of use указано: "AI может предлагать возможности на основе вашего профиля"
- Premium users согласны что их цели используются для matching

### Контроль
- Пользователи могут отключить проактивные сообщения (`/settings notifications`)
- Premium может установить лимит рекомендаций в день (default: 5)
- Все сообщения логируются для модерации

### Безопасность
- Персональные данные (email, phone) НЕ передаются
- Matching только по интересам/навыкам из публичного профиля
- AI не раскрывает "кто заказчик" рекомендации

## Метрики успеха

### Для мониторинга
- **Matching accuracy**: % релевантных совпадений
- **Response rate**: % ответов на рекомендации
- **Conversion rate**: % достижения целей Premium
- **User satisfaction**: feedback от получателей

### Цели
- Matching accuracy > 70%
- Response rate > 15%
- Conversion rate > 30%
- User satisfaction > 4.2/5

## Технические детали

### Зависимости
- `DeepSeek AI` для анализа целей и генерации сообщений
- `SQLAlchemy` для matching запросов
- `APScheduler` для фоновых jobs
- `aiogram` для отправки сообщений

### Performance
- Matching query: ~50ms для 10k пользователей (indexed fields)
- AI analysis: ~2-3s для извлечения целей
- AI generation: ~1-2s на сообщение
- Total cycle: ~15-30s для обработки одного Premium пользователя

### Масштабируемость
- Фоновые jobs не блокируют основной бот
- Batch обработка для множества Premium пользователей
- Rate limiting: макс 5 рекомендаций за раз (защита от спама)

## FAQ

**Q: Как часто запускается автоматизация?**  
A: Каждые 12 часов для каждого Premium пользователя.

**Q: Сколько рекомендаций отправляется за раз?**  
A: Максимум 5 за один цикл, чтобы избежать спама.

**Q: Можно ли отключить проактивные сообщения?**  
A: Да, через `/settings notifications off`.

**Q: Узнают ли пользователи что это для кого-то?**  
A: Нет, сообщения выглядят как естественные рекомендации AI.

**Q: Как Premium узнает о результатах?**  
A: Получает отчёт после каждого цикла автоматизации.

**Q: Что если цели не установлены?**  
A: Автоматизация не запускается, отчёт: "Нет целей, установи в /profile".
