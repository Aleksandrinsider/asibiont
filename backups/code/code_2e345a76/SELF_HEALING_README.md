# Система Самоисправления AI Агента 🤖🔧

## Обзор

Система самоисправления автоматически обнаруживает проблемы в работе AI агента и выполняет восстановление до рабочего состояния. Агент может самостоятельно откатывать код и конфигурацию без вмешательства человека.

## Основные возможности

### 🚨 Автоматическое обнаружение проблем
- Мониторинг здоровья системы каждые 60 секунд
- Анализ паттернов ошибок в реальном времени
- Отслеживание метрик производительности

### 🔄 Автоматическое восстановление
- Откат кода к предыдущей рабочей версии
- Откат конфигурации при проблемах
- Перезапуск сервисов при необходимости
- Масштабирование нагрузки

### 📊 Мониторинг и алерты
- Создание снимков системы при успешной работе
- Хранение истории версий (последние 10)
- Оповещения администратора при проблемах

## Архитектура

### Компоненты системы

1. **SelfHealingAgent** - основной класс системы
2. **Error Patterns** - паттерны для обнаружения проблем
3. **System Snapshots** - снимки состояния системы
4. **Recovery Actions** - действия восстановления

### Паттерны ошибок

| Тип ошибки | Серьезность | Порог | Действия восстановления |
|------------|-------------|-------|-------------------------|
| Таймауты ответов | Высокая | 10 за 5 мин | Перезапуск сервиса |
| Ошибки инструментов | Средняя | 5 за 10 мин | Откат кода |
| Проблемы с БД | Критическая | 3 за 2 мин | Перезапуск + алерт |
| Низкое качество | Средняя | 20 за 15 мин | Откат конфигурации |
| Утечка памяти | Высокая | 1 за 30 мин | Перезапуск сервиса |

## Использование

### Базовое использование

```python
from ai_integration.self_healing import get_self_healing_agent

# Получение экземпляра агента
healing_agent = get_self_healing_agent()

# Создание снимка рабочей системы
healing_agent.create_system_snapshot(is_working=True)

# Запуск мониторинга
healing_agent.start_monitoring()

# Остановка мониторинга
healing_agent.stop_monitoring()
```

### Ручное восстановление

```python
# Откат кода к последней рабочей версии
healing_agent._rollback_code()

# Откат конфигурации
healing_agent._rollback_config()

# Перезапуск сервиса
healing_agent._restart_service()
```

### CLI Утилита управления

Для удобного управления системой из командной строки используйте `self_healing_cli.py`:

## Мониторинг здоровья

### Метрики здоровья системы

- **Качество ответов**: средняя оценка ≥ 4/10
- **Время ответа**: среднее ≤ 5000 мс
- **Использование CPU**: ≤ 80%
- **Использование памяти**: ≤ 85%
- **Подключение к БД**: должно работать
- **Размер очереди логов**: ≤ 80% от лимита

### Уровни здоровья

- **🟢 Здоровая**: ≥ 70% проверок проходят
- **🟡 Предупреждение**: 40-70% проверок
- **🔴 Критическая**: < 40% проверок

## Безопасность и ограничения

### Меры безопасности

- **Ограничение откатов**: только к рабочим версиям
- **Валидация бэкапов**: проверка целостности перед восстановлением
- **Ограничение частоты**: предотвращение спама действиями восстановления
- **Изоляция**: действия восстановления не влияют на основной поток

### Ограничения

- Откатывает только Python файлы (.py)
- Не восстанавливает данные БД
- Требует ручного вмешательства при некоторых типах ошибок
- Зависит от доступности системных ресурсов

## Troubleshooting

### Проблема: Система не обнаруживает ошибки

**Решение:**
- Проверьте настройки паттернов ошибок
- Убедитесь, что мониторинг запущен
- Проверьте логи на наличие ошибок в системе мониторинга

### Проблема: Откат не работает

**Решение:**
- Проверьте наличие рабочих снимков: `len(agent.version_history)`
- Убедитесь в наличии бэкап файлов
- Проверьте права доступа к директориям бэкапов

### Проблема: Ложные срабатывания

**Решение:**
- Настройте пороги срабатывания паттернов
- Увеличьте временные окна анализа
- Исключите известные benign ошибки
