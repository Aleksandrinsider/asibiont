# AI Оптимизации для сокращения расходов

## 🎯 Цели оптимизации

1. **Сокращение расходов на ИИ** без потери качества ответов
2. **Улучшение производительности** агента
3. **Готовность к новым моделям** DeepSeek V3.2

## 🚀 Реализованные оптимизации

### 1. Кеширование ответов ИИ

**Что делает:**
- Кеширует ответы ИИ на 1 час для одинаковых запросов
- Использует MD5 хеширование для генерации ключей кеша
- Автоматически очищает устаревший кеш

**Где применяется:**
- Анализ профиля пользователя (`analyze_with_ai`)
- Классификация намерений (`classify_intent_cached`)

**Экономия:** До 60% запросов к ИИ для повторяющихся сценариев

### 2. Конфигурируемые параметры ИИ

```bash
# В .env файле
DEEPSEEK_MODEL=deepseek-chat  # или deepseek-v3.2
AI_CACHE_ENABLED=True
AI_MAX_TOKENS_RESPONSE=1000   # Ограничение токенов
AI_MAX_TOKENS_ANALYSIS=500
AI_TEMPERATURE_LOW=0.1        # Для точных ответов
AI_TEMPERATURE_HIGH=0.7       # Для креативных ответов
```

### 3. Интеллектуальный выбор температуры

- **Низкая температура (0.1)**: Для фактов, анализа, извлечения задач
- **Высокая температура (0.7)**: Для генерации идей, рекомендаций

## 📊 Метрики производительности

### До оптимизации:
- Каждый анализ профиля = 1 запрос к ИИ
- Классификация намерений = повторные вычисления
- Нет ограничений на токены

### После оптимизации:
- Анализ профиля = кешируется на 1 час
- Классификация намерений = LRU кеш 100 элементов
- Ограничение токенов предотвращает перерасход

## 🔄 Обновление до DeepSeek V3.2

### Как обновить модель:

1. **Измените переменную окружения:**
```bash
export DEEPSEEK_MODEL=deepseek-v3.2
```

2. **Или в .env файле:**
```
DEEPSEEK_MODEL=deepseek-v3.2
```

3. **Для специальной версии:**
```
DEEPSEEK_MODEL=deepseek-v3.2-speciale
```

### Преимущества V3.2:
- **Reasoning-first**: Лучше для сложных задач
- **Экономичнее**: Оптимизированная архитектура
- **Быстрее**: Улучшенная скорость генерации

## 💡 Дополнительные возможности оптимизации

### 1. Пакетная обработка
```python
# Группировать похожие запросы
batch_requests = [
    {"prompt": "Анализ задачи 1", "priority": "high"},
    {"prompt": "Анализ задачи 2", "priority": "low"}
]
```

### 2. Умное кеширование по контексту
- Кеширование по пользователю
- Кеширование по типу задачи
- Кеширование по времени суток

### 3. Fallback стратегии
- Если ИИ недоступен → использовать кеш
- Если кеш пуст → использовать правила
- Если правила не сработали → минимальный ответ
