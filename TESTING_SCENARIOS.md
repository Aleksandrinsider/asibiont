# Тестовые сценарии для проверки возможностей агента

## Цель тестирования
Проверить что агент умеет понимать запросы пользователя и добавлять, изменять или удалять данные из профиля, БД, Redis и других мест.

---

## 1. Работа с ПРОФИЛЕМ

### ✅ Доступные операции:
- **Добавление данных** к существующим полям (навыки, интересы, цели)
- **Изменение данных** (город, компания, должность, био, языки, часовой пояс)
- **Удаление данных** (префикс `-` для списков, пустая строка `''` для полной очистки)

### Тестовые команды:

#### Добавление к профилю:
```
Добавь в мои интересы футбол и программирование
```
**Ожидаемый результат:** Агент вызовет `update_profile(interests="футбол, программирование")`, интересы добавятся к существующим.

#### Изменение профиля:
```
Я работаю в компании TechCorp на должности Senior Developer
```
**Ожидаемый результат:** Агент вызовет `update_profile(company="TechCorp", position="Senior Developer")`.

#### Удаление из списка:
```
Убери из моих интересов криптовалюты
```
**Ожидаемый результат:** Агент вызовет `update_profile(interests="-криптовалюты")`.

#### Полная очистка поля:
```
Удали все мои интересы
```
**Ожидаемый результат:** Агент вызовет `update_profile(interests="")`.

#### Изменение города:
```
Я переехал в Москву
```
**Ожидаемый результат:** Агент вызовет `update_profile(city="Москва")`.

---

## 2. Работа с ЗАДАЧАМИ

### ✅ Доступные операции:
- **Создание задачи** (`add_task`)
- **Удаление задачи** (`delete_task`)
- **Удаление всех задач** (`delete_all_tasks`)
- **Завершение задачи** (`complete_task`)
- **Установка напоминания** (`set_reminder`)
- **Установка приоритета** (`set_priority`)

### Тестовые команды:

#### Создание задачи:
```
Напомни мне завтра в 10:00 позвонить маме
```
**Ожидаемый результат:** Агент вызовет `add_task(title="Позвонить маме", reminder_time="2026-01-17 10:00")`.

#### Удаление конкретной задачи по названию:
```
Удали задачу "Позвонить маме"
```
**Ожидаемый результат:** Агент вызовет `delete_task(task_title="Позвонить маме")`.

#### Удаление по ID:
```
Удали задачу 123
```
**Ожидаемый результат:** Агент вызовет `delete_task(task_id=123)`.

#### Завершение задачи:
```
Я позвонил маме, задача выполнена
```
**Ожидаемый результат:** Агент вызовет `complete_task(task_title="Позвонить маме")`.

#### Удаление всех задач (с подтверждением):
```
Удали все мои задачи
```
**Ожидаемый результат:** 
1. Агент запросит подтверждение: "Ты точно хочешь удалить ВСЕ задачи? Это действие нельзя отменить."
2. После подтверждения вызовет `delete_all_tasks()`.

## 3. Работа с КОНТЕКСТОМ (Redis)

### ✅ Доступные операции:
- **Сохранение информации в долговременную память** (`update_user_memory`)
- Контекст диалога автоматически сохраняется в Redis (последние 50 сообщений, TTL 24 часа)

### Тестовые команды:

#### Сохранение предпочтений:
```
Запомни, я предпочитаю работать утром с 8 до 12
```
**Ожидаемый результат:** Агент вызовет `update_user_memory(info="Предпочитает работать утром с 8 до 12")`.

#### Сохранение привычек:
```
Каждый понедельник я хожу на йогу в 19:00
```
**Ожидаемый результат:** Агент вызовет `update_user_memory(info="Каждый понедельник йога в 19:00")`.

## 4. Работа с ДЕЛЕГИРОВАНИЕМ

### ✅ Доступные операции:
- **Делегирование задачи другому пользователю** (`delegate_task`)
- Доступно только для тарифов **Silver** и **Gold**
- Bronze пользователи могут **получать** делегированные задачи

### Тестовые команды:

#### Делегирование задачи:
```
Делегируй @sportfan2 задачу подготовить отчет до завтра 15:00
```
**Ожидаемый результат:** Агент вызовет `delegate_task(title="Подготовить отчет", delegated_to_username="sportfan2", reminder_time="завтра в 15:00")`.

#### Попытка делегирования Bronze пользователем:
```
(Пользователь с Bronze тарифом) Делегируй @sportfan3 задачу
```
**Ожидаемый результат:** Агент вернет ошибку: "🥉 Делегирование задач доступно только на тарифах Серебро и Золото."

## 5. Комплексные сценарии

### Сценарий 1: Обновление профиля + создание задачи
```
Я работаю в Google на должности Product Manager. Напомни мне завтра в 9:00 подготовить презентацию для команды.
```
**Ожидаемый результат:**
1. `update_profile(company="Google", position="Product Manager")`
2. `add_task(title="Подготовить презентацию для команды", reminder_time="2026-01-17 09:00")`

### Сценарий 2: Изменение профиля + удаление старых данных
```
Я больше не занимаюсь криптовалютами. Теперь мои интересы - AI и машинное обучение.
```
**Ожидаемый результат:**
1. `update_profile(interests="-криптовалюты")`
2. `update_profile(interests="AI, машинное обучение")`

### Сценарий 3: Создание + завершение + удаление задач
```
1. Создай задачу "Купить молоко" на сегодня 18:00
2. (После некоторого времени) Я купил молоко
3. Удали задачу "Купить молоко"
```
**Ожидаемый результат:**
1. `add_task(title="Купить молоко", reminder_time="2026-01-16 18:00")`
2. `complete_task(task_title="Купить молоко")`
3. `delete_task(task_title="Купить молоко")` (если задача уже завершена, удалит из архива)

## 6. Проверка через UI

### Тесты через веб-интерфейс:

1. **Откройте дашборд:** http://localhost:8080/direct_login?user_id=146333757

2. **Проверьте профиль:**
   - Отправьте: "Добавь в мои навыки Python и JavaScript"

   - Все действия (завершение, удаление, делегирование, скрытие контакта) должны показывать уведомления в правом верхнем углу
   - Уведомления об ошибках тарифа должны содержать кнопку "Повысить тариф"

## 7. Структура данных

### Профиль (UserProfile в БД PostgreSQL/SQLite):
- `skills` - навыки (список через запятую)
- `interests` - интересы (список через запятую)
- `goals` - цели (список через запятую)
- `city` - город
- `company` - компания
- `position` - должность
- `bio` - сфера деятельности
- `languages` - языки
- `current_plans` - текущие планы
- `timezone` - часовой пояс

### Задачи (Task в БД):
- `title` - название
- `description` - описание
- `reminder_time` - время напоминания
- `due_date` - дедлайн
- `priority` - приоритет
- `status` - статус (active/completed)
- `delegated_to_username` - кому делегирована

### Контекст (Redis):
- Ключ: `context:{user_id}`
- Значение: JSON массив последних 50 сообщений
- TTL: 24 часа
- Структура: `[{"user": "...", "agent": "...", "timestamp": "..."}]`

### Память (UserMemory в БД):
- `info` - сохраненная информация о пользователе
- `created_at` - время создания записи

## Дополнительные проверки

### Проверка целостности данных:
1. Создайте задачу → проверьте что она в БД → удалите → проверьте что её нет
2. Обновите профиль → перезагрузите страницу → проверьте что данные сохранились
3. Добавьте интерес → добавьте ещё один → проверьте что оба сохранены
4. Очистите поле → проверьте что оно действительно пустое

### Проверка Redis:
```bash
# Подключитесь к Redis (если локально)
redis-cli

# Проверьте ключи контекста
KEYS context:*

# Посмотрите контекст конкретного пользователя
GET context:146333757

# Проверьте TTL
TTL context:146333757
```

### Проверка БД:
```sql
-- Проверьте профиль
SELECT * FROM user_profiles WHERE user_id = (SELECT id FROM users WHERE telegram_id = 146333757);

-- Проверьте задачи
SELECT * FROM tasks WHERE user_id = (SELECT id FROM users WHERE telegram_id = 146333757);

-- Проверьте память
SELECT * FROM user_memory WHERE user_id = (SELECT id FROM users WHERE telegram_id = 146333757);
```
