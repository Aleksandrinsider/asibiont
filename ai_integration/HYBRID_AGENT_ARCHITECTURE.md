# Гибридный Автономный AI-Агент

## 🎯 Концепция

Новый **HybridAutonomousAgent** объединяет лучшее из двух миров:
- **Автономность AI**: агент сам планирует стратегию выполнения запроса
- **Надежность инструментов**: использует проверенные handlers вместо генерации кода

## 🏗️ Архитектура

### Трехэтапный процесс обработки запроса

```
Запрос → [1. Планирование] → [2. Выполнение] → [3. Рефлексия] → Ответ
```

#### 1️⃣ Планирование стратегии (`plan_strategy`)

AI анализирует запрос пользователя и создает **план действий**:

```python
{
    "intent": "создать задачу",
    "needs_context": false,
    "actions": [
        {
            "tool": "add_task",
            "params": {"title": "Купить молоко", "reminder_time": "завтра в 9:00"},
            "reason": "пользователь хочет создать задачу"
        }
    ],
    "response_strategy": "подтвердить создание задачи с деталями"
}
```

**Преимущества**:
- AI понимает намерение пользователя
- Выбирает правильные инструменты
- Не генерирует код - использует готовые handlers

#### 2️⃣ Выполнение действий (`execute_actions`)

Агент выполняет запланированные действия через **готовые handlers**:

```python
from . import handlers

result = await handlers.add_task(
    user_id=user_id,
    title="Купить молоко",
    reminder_time="завтра в 9:00"
)
```

**Преимущества**:
- Быстро - не нужно генерировать код
- Надежно - используются проверенные функции
- Безопасно - нет выполнения сгенерированного кода

#### 3️⃣ Рефлексия и ответ (`reflect_and_respond`)

AI анализирует результаты выполнения и формирует **естественный ответ**:

```
✅ Отлично! Я создал задачу «Купить молоко» с напоминанием на завтра в 9:00 утра.
Напомню тебе за 30 минут до этого времени.

📝 Детали задачи:
- Что: Купить молоко
- Когда: Завтра, 6 февраля, в 09:00
- Напоминание: 8:30 (за 30 минут)

Хочешь добавить что-то к этой задаче?
```

**Преимущества**:
- Дружелюбный тон
- Конкретные детали
- Полезные советы и вопросы

## 🆚 Сравнение подходов

### Tools-Based (старый)

```
Запрос → AI + Tools → Вызов handlers → Ответ
```

❌ **Недостатки**:
- AI должен сразу выбрать правильный tool
- Нет планирования
- Нет рефлексии над результатами

### Autonomous (старый)

```
Запрос → AI генерирует код → Выполнение кода → Ответ
```

❌ **Недостатки**:
- Медленно (генерация кода)
- Ненадежно (код может быть некорректным)
- Шаблонные ответы

### Hybrid (новый) ✅

```
Запрос → [Планирование] → [Выполнение handlers] → [Рефлексия] → Ответ
```

✅ **Преимущества**:
- Автономное планирование
- Быстрое выполнение через handlers
- Естественные ответы с рефлексией
- Адаптация к ошибкам

## 🔧 Доступные инструменты

Агент имеет доступ к следующим handlers:

| Инструмент | Описание |
|-----------|----------|
| `add_task` | Создать новую задачу |
| `list_tasks` | Показать список задач |
| `complete_task` | Отметить задачу как выполненную |
| `reschedule_task` | Перенести задачу |
| `delete_task` | Удалить задачу |
| `edit_task` | Редактировать задачу |
| `get_task_details` | Получить детали задачи |
| `find_relevant_contacts_for_task` | Найти контакты для активности |
| `delegate_task` | Делегировать задачу |
| `show_profile` | Показать профиль |
| `update_profile` | Обновить профиль |
| `check_subscription_status` | Проверить подписку |

## 🎓 Ключевые принципы

1. **AI планирует, код выполняет** - AI выбирает стратегию, handlers делают работу
2. **Рефлексия обязательна** - AI анализирует результаты и формирует ответ
3. **Естественность общения** - дружелюбный тон, конкретные детали, полезные советы
4. **Минимум вызовов** - планируем минимальное количество действий
5. **Контекст важен** - учитываем предыдущие сообщения и текущие задачи
